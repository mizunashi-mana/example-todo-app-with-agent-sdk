# ツールコーリング非対応モデルの警告表示

## 目的・ゴール

phi4-mini など、Ollama のネイティブツールコーリングに非対応のモデルを選択した際に、
チャットパネルに警告を表示する。

## 背景

- phi4-mini は Ollama API の `tool_calls` フィールドを使わず、テキストとして JSON を返す
- 現状のコードは `result.toolCalls.length > 0` でツールコール検出するため、phi4-mini では常にテキスト応答になる
- ユーザーが phi4-mini を選択してツールコーリングが動かないのは混乱を招く

## 実装方針

- モデルがツールコール非対応の場合を検出し、UI に警告を表示する
- 最小変更で対応する

## 完了条件

- [ ] ツールコーリング非対応モデル選択時に警告が表示される
- [ ] granite4 など対応モデルでは警告が表示されない
- [ ] lint が通る
- [ ] 既存テストが通る

## 作業ログ

- Ollama API で直接テスト: phi4-mini は `tool_calls` ではなくテキストで JSON を返すことを確認
- granite4 は正常に `tool_calls` フィールドを返すことを確認
